Inventory Comment Pattern Checker v5

Verbose analysis edition. This page reviews an inventory export, explains exactly what facts were expected from each row, compares those facts against the inventory comment, and separates true contradictions from missing-but-reviewable details.

What this verbose version adds

  • Broader laptop/tablet analysis: extracts brand, model family, CPU, CPU speed, RAM, storage, screen size, color, model codes, and network hints from description, itemnumber, mfgr, and related columns.
  • More transparent scoring: every row gets a score based on comparable facts, plus a row narrative explaining what was confirmed, missing, or contradictory.
  • Verbose dashboards: includes data profile, severity mix, reason breakdown, fact coverage, comment-quality signals, warehouse/condition breakdowns, and top repeated item numbers.
  • Less noisy mismatches: optional attributes such as color and screen size are treated as useful context, but they do not fail a row unless there is a clear contradiction.
  • Downloadable detail: exported review CSV now includes description, serial, warehouse, location, parsed facts, detailed checks, and row narrative.
Required headers: itemid, itemnumber, inventorycomments. Strongly recommended: description, mfgr, serialnumber, conditioncode, warehouse, location, status, unitcost.
Leave blank to scan every row in the CSV. Matching is case-insensitive and checks item number, item ID, and serial number.
Recommended mode keeps the table focused. Use Strict or Audit when you want to inspect missing details and scoring logic.
